# DeepVCF 诊断报告反馈功能 PRD

## 1. 概述

### 1.1 背景

DeepVCF 是 ONEX Intelligence 基于 ACMG/AMP 指南的基因变异致病性智能评估平台。当前报告页面为只读展示，缺乏医生对 AI 诊断质量的结构化反馈通道。

在早期临床试用中，医生反馈指出：AI 在文献内容理解与证据提取层面存在短板——文献检索的语义关联效果尚可，但对文献内容的解读和证据采信不够准确；在表型特异性较低的病例（如"癫痫"）中，基因定位能力下降。这些问题需要精细化的反馈数据来定位和优化。

### 1.2 目标

在现有报告结果页中嵌入三级反馈机制（证据级 → 变异级 → 报告级），让医生在审阅报告的自然流程中顺手标注，形成数据闭环，精准定位模型在检索、理解、推理各环节的问题。

### 1.3 成功指标

- 反馈提交率 ≥ 30%（有反馈的报告数 / 总报告数）
- 单次反馈平均耗时 < 3 分钟
- 反馈数据能区分"检索错误 vs 理解错误 vs 推理错误"三类证据问题

---

## 2. 用户与场景

### 2.1 目标用户

临床遗传科医生、分子诊断医生——在审阅 DeepVCF 生成的 ACMG 诊断报告后，对 AI 结果进行专业验证。

### 2.2 典型使用场景

医生打开某患者的诊断报告，从上到下审阅变异详情。审阅过程中，对某条 ACMG 证据（如 PVS1）的文献解读有异议，随手点击"有误"并勾选"文献相关但内容解读有误"。继续审阅完其他变异后，滚动到页面底部，确认 AI 首位诊断命中了临床诊断，给四个维度打分，点击提交。全程不离开报告页面，额外耗时约 1-2 分钟。

## 3. 现有页面结构

报告页面从上到下依次为：

**顶部导航栏** — 蓝色背景，平台名称与副标题，左上角返回按钮。

**操作栏** — 右侧「导出 PDF」「导出 Word」按钮。

**患者信息概览** — 白色卡片，表格式布局展示患者基本信息（ID、年龄、性别、起病年龄、检测类型、家系结构），下方为表型说明文本。

**主要变异摘要表** — 「可能解释患者主要表型的变异」区域，包含基因、位置、变异信息、来源、疾病与遗传方式、致病性 badge、证据列表。附「结果说明」卡片。

**变异详情卡片（可折叠）** — 每个变异一张卡片，包含浅蓝色卡片头、基本信息区、ACMG 评分总览（灰底）、致病性证据区（浅黄色可折叠条目，含证据规则、判定依据、参考文献）、良性证据区、相关疾病卡片。

**意外发现的变异** — 结构同主要变异，badge 为橙色"可能致病"。

**其余供参考的变异** — 结构同上，额外含群体频率数据。

**综合分析结论** — 检测结果总结、ACMG 分类阐述、临床影响、参考文献。

## 4. 新增功能设计

### 4.1 证据级别反馈（A 层）

**位置：** 每条 ACMG 证据条目（如 PVS1、PM6、PP3 等）展开内容的底部。

**功能描述：**

医生审阅每条证据时，底部显示三态按钮组（pill 样式）：「👍 正确」「👎 有误」「➖ 不确定」。选中态分别为绿色、红色、灰色高亮。

点击「有误」后平滑展开详细标注区域，包含：

- **问题类型（checkbox 可多选）** — 证据强度判定有误（如该 Moderate 标成了 Strong）/ 引用文献不相关 / 文献相关但内容解读有误 / 该证据不应被采用
- **补充说明（选填）** — 自由文本框
- **PMID 补充（选填）** — 供医生提供更准确的参考文献

点击「正确」或「不确定」时，若详细区域已展开则收起。

**折叠态指示：** 已标注的证据在折叠状态下，标题行分数 badge 旁显示一个小圆点（绿/红/灰），表示已有反馈。

**数据价值映射：**

| 问题类型选项 | 对应优化的模型模块 |
|---|---|
| 引用文献不相关 | 语义检索模型 |
| 文献相关但内容解读有误 | 文献理解模型 |
| 证据强度判定有误 | ACMG 推理模型 |
| 该证据不应被采用 | 证据筛选逻辑 |
| 补充的 PMID | 检索训练数据扩充 |

### 4.2 变异级别反馈（B 层）

**位置：** 每个变异卡片的 ACMG 评分总览区域下方。

**功能描述：**

显示问题「您是否同意该变异的致病性分类？」，三态按钮组：「✓ 同意」「✗ 不同意」「— 不确定」。

点击「不同意」后展开：

- **正确分类（radio 单选）** — 致病 (P) / 可能致病 (LP) / 意义不明 (VUS) / 可能良性 (LB) / 良性 (B)
- **简要理由（选填）** — 自由文本框

点击「同意」或「不确定」时收起展开区。

**数据价值：** 构建人工标注的致病性判定数据集，直接回馈 ACMG 分类模型。

### 4.3 报告级别反馈（C 层）

**位置：** 页面底部，综合分析结论卡片之后，新增独立反馈卡片。

**功能描述，分四个区块：**

**区块 1 — 诊断命中验证（必填）**

「AI 首位诊断是否命中您的临床诊断？」Radio 单选：首位命中 / 在 Top 3 内 / 在 Top 5 内 / 未命中。选择「未命中」后展开输入框，填写正确诊断（疾病名称或 OMIM ID）。

**区块 2 — 关键能力评分（选填）**

四个维度各用 1-5 星评分：表型解析与基因定位 / 变异位点发现能力 / 文献检索相关性 / 文献理解与证据提取质量。

**区块 3 — 补充反馈（选填）**

自由文本框，用于描述结构化选项无法覆盖的具体场景（如"test2 的文献引用正确但对特定段落的理解有误"）。

**区块 4 — 上传参考报告（选填）**

拖拽/点击上传区域，支持 PDF / Word / 图片，供医生上传临床确诊报告或已有遗传报告作为 ground truth。

底部「提交反馈」按钮，一次性收集 A + B + C 三层所有已标注内容统一提交。

### 4.4 悬浮反馈入口（D 层）

**位置：** 页面右下角固定悬浮。

**功能描述：** 显示当前反馈进度（如「已标注 3/8 条证据」），点击后平滑滚动到页面底部的报告级别反馈区域。

## 5. 交互规则

### 5.1 低摩擦原则

- 证据级和变异级反馈全部单击/两击完成，不强制填写，随用随标
- 报告级仅"诊断命中验证"为必填（前提是用户选择提交反馈），其余全部选填
- 所有反馈内嵌报告页面对应位置，不使用弹窗或独立页面

### 5.2 状态可见性

- 已标注的证据在折叠态通过彩色圆点指示反馈状态
- 悬浮按钮实时显示已标注进度
- 提交后报告页面显示「已反馈」标记

### 5.3 可修改性

- 提交前：所有反馈可随时修改
- 提交后：医生可重新进入报告回看和修改已提交的反馈

## 6. 视觉规范

### 6.1 色彩

- 主色调蓝色 #1a56db（标题、主要按钮、链接）
- 证据条目背景浅黄色 #fef9e7
- 致病 badge 红色 #dc2626，可能致病 badge 橙色 #f97316
- 变异卡片头部浅蓝色 #eff6ff

### 6.2 反馈组件样式

- 沿用现有 UI 的圆角（rounded-lg）、阴影（shadow-sm）、间距风格
- 反馈区域通过左侧 3px 彩色竖线与正文做视觉区分（蓝色 = 未标注，绿色 = 正确，红色 = 有误，灰色 = 不确定）
- 三态按钮组为 pill 样式，未选中态浅灰边框，选中态填充对应颜色
- 星级评分：已选金色，未选灰色

---

## 7. 设计原则

**内嵌而非打断** — 反馈融入审阅流程，不增加额外步骤。

**每个字段有明确的模型优化指向** — 不收集无用数据，每个反馈字段直接映射到模型 pipeline 的具体环节。

**结构化优先，开放补充** — 用预设选项覆盖高频问题类型，用自由文本捕捉长尾场景。

**渐进式披露** — 默认只显示最简反馈（三态按钮），详细标注仅在需要时展开，降低视觉负担。
