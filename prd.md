# VCF 增加反馈功能

## 一、背景与目标

**现状：** 当前 VCF 报告页面为只读展示，医生审阅 AI 生成的变异致病性评估结果后，缺乏对 AI 诊断质量进行反馈的通道。

**目标：** 在现有报告结果页中嵌入三级反馈机制（证据级 → 变异级 → 报告级），让医生在审阅过程中顺手标注，形成"AI 输出 → 医生反馈 → 模型优化"的数据闭环，持续反哺系统效果。

**核心价值：**
- 对医生：提供表达专业判断的出口，增强对工具的掌控感
- 对产品：获取高质量标注数据，量化 AI 评估准确率
- 对算法：积累真实临床反馈，驱动模型迭代优化

---

## 二、功能范围

本期需求包含以下三个层级的反馈能力：

| 层级 | 反馈对象 | 说明 |
|------|----------|------|
| 证据级反馈 | 单条 ACMG 证据项 | 针对证据准确性、强度判定进行反馈 |
| 文献级反馈 | 证据内的单条文献 | 针对单个文献的引用正确性、相关性、理解正确性等进行反馈 |
| 变异级反馈 | 单个变异的致病评级 | 针对 AI 给出的致病性分类（P/LP/VUS/LB/B）进行反馈 |
| 报告级反馈 | 整份报告 | 对诊断结论命中率、关键能力维度及整体质量进行评价 |

---

## 三、功能详细说明

### 3.1 变异致病评级反馈

#### 反馈入口

在每个变异的致病评级展示区域旁，增加反馈入口（如"反馈"按钮或图标），点击后展开反馈表单。

#### 反馈字段定义

**一级选项（必填，单选）：**

| 选项 | 英文标识 | 触发二级字段 |
|------|----------|--------------|
| 同意 | Agree | 无 |
| 不同意 | Disagree | 致病评级选项（必填）|
| 不确定 | Uncertain | 文本输入框（选填）|

**二级字段说明：**

- 当用户选择"不同意"时，需进一步选择其认为正确的致病评级（单选，必填）：

| 选项 | 英文标识 |
|------|----------|
| 致病 | Pathogenic (P) |
| 可能致病 | Likely Pathogenic (LP) |
| 意义不明 | VUS |
| 可能良性 | Likely Benign (LB) |
| 良性 | Benign (B) |

- 当用户选择"不确定"时，展示文本输入框，供用户输入理由（选填，限长 1000 字符）。

#### 提交反馈

1. 用户选择一级选项后，界面展开对应的二级字段，同时显示"提交"与"取消"按钮
2. 提交按钮状态：所有必填项填写完成后高亮可点击，否则置灰不可点击
3. 用户点击"提交"后：
   - a. 系统保存反馈数据；
   - b. 界面切换为已提交状态，显示"您的反馈已提交"提示文案；提示下方展示本次反馈详情（所选选项及填写内容）；展示"修改"按钮
4. 用户点击"取消"：收起反馈表单，恢复为未反馈状态，已填写内容不保留

#### 修改反馈

1. 用户点击"修改"按钮，反馈组件回到编辑态，回填上次提交的内容
2. 编辑态交互逻辑与首次提交一致
3. 用户修改后点击"提交"，覆盖上一次反馈记录
4. 用户点击"取消"，则不保存本次修改，界面恢复展示上次提交的反馈结果

### 3.2 证据级别反馈

#### 反馈入口

在每条 ACMG 证据项最底部，展示 `正确` `有误` 选项。

#### 反馈字段定义

- 点击 **正确**：直接记录为正向反馈，无需展开二级字段。
- 点击 **有误**：弹出轻量反馈面板，展示以下选项（多选，必填）：

| 选项 | 英文标识 |
|------|----------|
| 证据强度有误 | wrong_strength |
| 证据引用的文献不相关 | irrelevant_literature |
| 文献相关，但对文献内容的解读有误 | misinterpretation |
| 该证据不应被采用 | should_not_apply |
| 其他 | other |

- 当用户选择"其他"时，展开文本输入框（选填，限长 1000 字符）。

#### 提交反馈

1. 用户选择一级选项后，界面展开对应的二级字段，同时显示"提交"与"取消"按钮
2. 提交按钮状态：所有必填项填写完成后高亮可点击，否则置灰不可点击
3. 用户点击"提交"后：
   - a. 系统保存反馈数据；
   - b. 界面切换为已提交状态，显示"您的反馈已提交"提示文案；提示下方展示本次反馈详情（所选选项及填写内容）；展示"修改"按钮
4. 用户点击"取消"：收起反馈表单，恢复为未反馈状态，已填写内容不保留

#### 修改反馈

1. 用户点击"修改"按钮，反馈组件回到编辑态，回填上次提交的内容
2. 编辑态交互逻辑与首次提交一致
3. 用户修改后点击"提交"，覆盖上一次反馈记录
4. 用户点击"取消"，则不保存本次修改，界面恢复展示上次提交的反馈结果

### 3.3 文献引用反馈

#### 反馈入口

在每条关联文献的展示区域旁，放置"👍 / 👎"按钮。

#### 反馈字段定义

**一级选项（单选）：**

| 选项 | 触发二级字段 |
|------|--------------|
| 👍 | 无 |
| 👎 | PMID 补充输入框（选填）|

当用户标注"👎"时，展示反馈弹窗：

**原因标签（多选，至少选一项）：**

| 选项 | 英文标识 | 说明 |
|------|----------|------|
| 文献不相关 | not_relevant | 文献与当前变异/疾病无关 |
| 文献已过时 | outdated | 文献发表时间过早，结论已被更新研究取代 |
| 来源可疑 | questionable_source | 文献来源期刊/数据库可信度不足 |
| 文献信息有误 | incorrect_information | 文献内容被错误引用或解读 |
| 有更好的文献可替代 | better_source_available | 存在更合适的文献支持该证据 |

**补充说明（选填）：** 文本输入框，限长 1000 字符，供用户补充具体描述。
- 当用户选择"有更好的文献可替代"时，展示补充说明输入框，底纹：在此填写推荐文献的 PMID 或链接，帮助我们更好地优化

#### 提交反馈

1. 当用户标注"👍"时，自动保存反馈
2. 当用户标注"👎"时：
   - a. 界面展开对应的二级字段，同时显示"提交"与"取消"按钮
   - b. 提交按钮状态：用户选择了一个选项后，提交按钮高亮可点击，否则置灰不可点击
   - c. 用户点击提交按钮：
     - i. 系统保存反馈数据；
     - ii. 用户选择的选项，显示为高亮状态
   - d. 用户点击"取消"：收起反馈表单，恢复为未反馈状态，已填写内容不保留

#### 修改反馈

文献类型的反馈，无需"修改"按钮。

用户提交反馈后，👍 / 👎 按钮保持高亮显示当前选择状态，用户可随时点击另一侧按钮直接切换：
- 点击 👍：直接覆盖为正向反馈，👍 高亮
- 点击 👎：重新展开反馈弹窗，用户重新选择原因后提交，👎 高亮，原反馈记录覆盖

### 3.4 报告级反馈

报告级反馈区域位于报告页面底部，作为统一的整体评价入口。

#### 开放反馈

在评分区域下方设置一个自由文本框，供用户补充结构化选项无法覆盖的信息。

字段说明：提示文案为"请补充您的其他反馈"，限长 2000 字符，选填。

#### 提交与修改

1. 提交按钮状态：反馈输入框不为空时，提交按钮高亮展示
2. 用户点击"提交"后：
   - a. 系统保存反馈数据；
   - b. 界面切换为已提交状态，显示"您的反馈已提交"提示文案；展示"修改"按钮
3. 用户点击"取消"：收起反馈表单，恢复为未反馈状态，已填写内容不保留

#### 修改反馈

1. 用户点击"修改"按钮，反馈组件回到编辑态，回填上次提交的内容
2. 编辑态交互逻辑与首次提交一致
3. 用户修改后点击"提交"，覆盖上一次反馈记录
4. 用户点击"取消"，则不保存本次修改，界面恢复展示上次提交的反馈结果
