# 设计规范 - 开发侧适配版（第二版）

> 本文档给**产品/设计同学**使用：用 AI 生成页面与代码时，按本规范描述，生成的产出可直接交给开发落地，并与现有项目风格、组件库一致。  
> 规范中的颜色、圆角、间距等均以 **Design Token** 形式约定，便于后续统一改版。

**和开发同学的约定**：生成代码时请使用项目统一的**基础组件库**（Button、Card、Input、Tabs 等从组件库取），不要用 Tailwind 从零拼组件，也不要直接写 antd。这样样式和后续改版会由组件库统一控制。

---

## 一、设计理念与总体风格（与初稿一致）

- **行业属性**：医疗健康、AI 辅助诊断。
- **视觉关键词**：专业、严谨、现代、清晰、信任感。
- **整体风格**：现代 SaaS 风格，大面积浅灰/纯白背景，柔和阴影与圆角；清晰视觉层级与高对比度主题色，保证高密度数据可读性。

---

## 二、色彩规范（Design Token）

以下用 **Token 名称 + 色值** 约定，设计描述或给 AI 的提示里可引用名称或直接写色值；开发会在组件库中统一配置，便于全站一致和后续改版。

### 2.1 品牌主色

| Token 名称 | 色值 | 用途 |
|------------|------|------|
| 主色 | `#2563eb` | 主按钮、激活状态、强调信息 |
| 主色 Hover | `#1d4ed8` | 主按钮 Hover、导航激活 |
| 主色浅底 | `#eff6ff` | 选中卡片背景、标签背景等 |

### 2.2 中性色（背景、文字、边框）

| Token 名称 | 色值 | 用途 |
|------------|------|------|
| 页面/输入框背景 | `#f8fafc` | 页面全局背景、输入框默认背景 |
| 卡片/导航背景 | `#ffffff` | 卡片、导航栏、弹窗/抽屉 |
| 主标题/强调文字 | `#0f172a` | 大标题、强调文本 |
| 正文 | `#0f172a` | 模块标题、正文 |
| 次要文字 | `#64748b` | 描述、表单提示 |
| 辅助/禁用文字 | `#94a3b8` | 占位符、时间戳、禁用 |
| 标准边框/分割线 | `#e2e8f0` | 卡片轮廓、分割线 |
| 浅分割线 | `#f1f5f9` | 内部分割线 |

### 2.3 语义色

| 含义 | 色值 | 用途 |
|------|------|------|
| 成功/完成 | `#22c55e` / `#059669` | 已完成步骤、高置信度、成功提示 |
| 警告/高亮 | `#f59e0b`（配浅底） | 关键词高亮、警告 |
| 危险/重要 | `#dc2626` / `#b91c1c` | 退出 Hover、重要强调 |
| AI/特殊 | `#9333ea` | AI 分析、HPO 等特殊节点 |

## 三、圆角、阴影、间距

| 类型 | 取值 | 用途 |
|------|------|------|
| 圆角 | 12px | 卡片、主按钮、输入框 |
| 大圆角 | 16px | 强调卡片、大面板 |
| 阴影 | 柔和 | 普通卡片 |
| 阴影 | 略强 | 强调卡片、弹窗 |
| 间距 | 8 / 16 / 24 / 32px | 模块内、模块间；大区块间可用 48px |

---

## 四、排版与字体

- **字体**：全局使用 Inter、system-ui、sans-serif；代码/专业术语（如 OMIM）用等宽字体。
- **层级**：
  - Hero 标题：大号、字重 800，字间距略紧。
  - 模块标题：大一号、字重 700。
  - 正文/表单：14px 或 16px，常规字重。
  - 辅助/标签：12px，medium 或 bold，英文可大写 + 加宽字间距。

以上由组件库统一配置，设计按层级描述即可。

---

## 五、布局与空间

- **页面框架**：
  - 顶部导航：高度 64px，白底，底部 1px 边框，左右留白 24px。
  - 主内容区：撑满剩余空间，背景用「页面/输入框背景」色，内容区可滚动。
- **内容容器**：居中定宽——首页约 1400px、列表约 1200px、流程页约 1152px；内边距响应式 16 / 24 / 32px。
- **模块间距**：区块间 24px 或 32px，大区块间 48px。

## 六、核心组件（视觉与用法）

描述效果即可；实现时请使用**基础组件库**的对应组件（Button、Card、Input、Tabs、Tag、Badge、Modal、Drawer、Spin 等），不要用 Tailwind 从零拼。

### 6.1 按钮

- **主按钮**：品牌主色背景、白字、圆角 12px、带轻微阴影；Hover 时颜色略深。
- **次要/幽灵**：浅灰底、灰字、细边框、圆角 8px；Hover 时背景略深。
- **导航/Tab**：默认灰字、Hover 浅灰底；选中时浅灰底 + 主色字、字重加粗。

### 6.2 卡片与容器

- **基础卡片**：白底、细边框、圆角 12px、柔和阴影。
- **强调卡片**（如首页输入区）：圆角 16px、阴影略强，顶部可加一条蓝–靛–紫渐变条；内部次级区块可用浅灰或主色浅底区分层级。

### 6.3 表单与输入

- **输入框/多行**：浅灰底、细边框、圆角 12px；聚焦时主色边框与轻微外发光。滚动条样式与主题一致（由组件库统一）。
- **单选/复选**：选项容器浅灰底、细边框；选中主色；Hover 主色浅底。

### 6.4 标签与徽标

- **状态徽标**：小尺寸、大圆角或全圆角、带背景色（如主色浅底 + 主色字）。
- **序号/排名**：小方或圆，Top1 用主色，其余用中性色。

---

## 七、交互与动画

- **过渡**：以组件自带的过渡为主（弹窗、抽屉、下拉等），节奏由组件库统一。
- **可自定义的范围**：页面整体淡入、列表/卡片 Hover 时背景或边框颜色变化，时长约 200ms 即可。
- **加载**：使用 Spin 或 Skeleton；若需「步骤条 + 百分比」等定制加载，由开发在组件库内实现。

避免在组件上叠加复杂的自定义动画，以免与组件库动效冲突。

---

## 八、图标

- **图标库**：统一使用 **@ant-design/icons**。
- **用法**：按钮内、标题旁、步骤条内，尺寸与背景容器由组件库统一（如小号图标 + 主色浅底圆形容器）。

---

## 九、给 AI 的提示要点（生成代码时）

1. 使用项目**基础组件库**的组件（Button、Card、Input、Tabs、Tag、Badge、Modal、Drawer、Spin 等），这些组件在技术上是基于 **Ant Design** 封装实现的；不要用 Tailwind 类从零拼，也不要直接写 antd。
2. 如果需要更个性化的视觉或交互，可以假设组件库支持按「全局主题」或「变体」来调整样式与功能，例如：  
   - **全局统一的改动**：通过主题 / Token 修改对应组件的默认样式，由组件库在 Ant Design 之上统一封装；  
   - **单页 / 单场景的改动**：优先使用组件库提供的 `variant` / `type` 等属性（如 `type=\"primary\"`、`variant=\"soft\"`）；若需要新增风格，可在设计中约定类似 `oVariant=\"variant1\"` 的变体名，由组件库在内部基于该变体名应用不同的 className / styles / 行为。
3. 颜色、圆角、间距按本规范中的 **Token 名称或色值** 描述；具体实现由组件库统一。
4. 新页面结构：顶部导航 + 主内容区（定宽容器 + 响应式内边距）。

## 十、与初稿的变动说明（给设计同学）

| 初稿写法 | 第二版写法 | 原因（简要） |
|----------|------------|--------------|
| 色彩/间距用 Tailwind 类名（如 bg-slate-50、rounded-xl） | 改为 Token 名称 + 色值/数值表 | 项目用组件库统一控制样式，Tailwind 与组件库易冲突；Token 便于统一改版。 |
| 排版用 text-4xl、font-bold 等 | 改为「字号与字重层级」描述 | 与组件库的字体体系一致，不在代码里写 Tailwind 字体类。 |
| 按钮/卡片用 Tailwind 写样式 | 改为「视觉描述 + 用组件库组件」 | 保证和现有封装一致，减少样式冲突。 |
| 动画用 transition-all、animate-fade-in 等 | 改为「以组件自带过渡为主，仅页面/Hover 轻量过渡」 | 组件库自带动效，自定义动画易冲突；轻量过渡更稳定。 |
| 未说明代码从哪取组件 | 明确：从**基础组件库**取，不用 Tailwind、不直接写 antd | 这样生成的代码可直接交给开发使用。 |

## 附录：给开发同学的技术说明（可选阅读）

以下供开发同学或需要对接实现时参考；设计同学可跳过。

- **依赖关系**：业务项目不直接依赖 antd，而是依赖**自研基础组件库**；该组件库基于 Ant Design 封装与定制（样式、Token）。AI 生成业务代码时应从**自研组件库** import 组件（包名由项目提供），不要直接 `import from 'antd'`。
- **Token 落点**：本规范中的色彩、圆角、阴影、间距、字体等 Token，在**自研基础组件库**层通过主题/ConfigProvider 配置并传递给底层 Ant Design；业务项目只使用组件库组件，不写死色值或重复配主题。
- **样式**：业务侧不在组件上堆叠 Tailwind 工具类；若需微调，在组件库内封装或通过 props/className 约定。
- **动画**：组件内置过渡由 Ant Design 的 motion 控制；仅在页面容器或外层包装上做轻量 transition（如淡入、Hover 背景/边框），避免 Tailwind 动画类与组件动效冲突。
